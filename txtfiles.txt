What you actually did is:

---

# 1. `git init`

```bash id="o44rba"
git init
```

### What happened?

You converted your normal HTML folder into a **Git repository**.

Git now starts tracking:

* `index.html`
* `style.css`
* `script.js`

A hidden `.git` folder gets created which stores version history.

---

# 2. `git add .`

```bash id="9vs1qd"
git add .
```

### What happened?

You told Git:

> “Prepare all files to be saved.”

This is called **staging**.

---

# 3. `git commit`

```bash id="ylk4c6"
git commit -m "Initial commit"
```

### What happened?

You created a **snapshot/version** of your project.

Think of commit like:

* Save point in a game
* Backup version of project

Git stores:

* what files changed
* when they changed
* commit message

---

# 4. Created GitHub Repository

On [GitHub](https://github.com?utm_source=chatgpt.com) you created an online repository.

### Why?

To store project online and share it.

---

# 5. `git remote add origin`

```bash id="tnj3wd"
git remote add origin <repo-link>
```

### What happened?

You connected:

* Local VS Code project
  with
* Online GitHub repository

`origin` means GitHub repo link.

---

# 6. `git push`

```bash id="kfh1yl"
git push -u origin main
```

### What happened?

You uploaded your local project files to GitHub.

So now:

* Files exist on your PC
* AND on GitHub online

---

# 7. `git branch feature`

```bash id="0j7q8n"
git branch feature
```

### What happened?

You created another working area called a **branch**.

Purpose:

* Test changes safely
* Main project stays unaffected

Like creating a duplicate copy to experiment.

---

# 8. `git checkout feature`

```bash id="1e5qt9"
git checkout feature
```

### What happened?

You switched from:

* `main`
  to
* `feature`

Now any changes happen only in feature branch.

---

# 9. Edited HTML File + Commit

You added some content and committed again.

### Meaning:

Git created another version history entry.

---

# 10. `git merge feature`

```bash id="d48t5u"
git merge feature
```

### What happened?

You combined:

* feature branch changes
  into
* main branch

So experimental changes become part of main project.

---

# 11. `git pull`

```bash id="lqv7go"
git pull origin main
```

### What happened?

Git checks GitHub for latest updates and downloads them to your PC.

Used when:

* Another person changed files
* Or you changed from another device

---

# Overall Concept

You basically learned:

| Operation | Meaning                   |
| --------- | ------------------------- |
| `init`    | Start Git tracking        |
| `add`     | Prepare files             |
| `commit`  | Save version              |
| `push`    | Upload to GitHub          |
| `pull`    | Download latest version   |
| `branch`  | Create separate workspace |
| `merge`   | Combine branches          |

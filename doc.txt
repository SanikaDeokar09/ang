# Docker Container Environment — Viva Explanation

---

# Step 1: Create Dockerfile

Command:

```bash id="g4m8qx"
notepad Dockerfile
```

Inside file:

```dockerfile id="p2v7wl"
FROM ubuntu
CMD ["echo", "Docker Container Environment Created"]
```

### Explanation

This file contains instructions for Docker.

* `FROM ubuntu`
  → Use Ubuntu Linux as base environment

* `CMD [...]`
  → When container starts, print the message

Dockerfile acts like a recipe.

---

# Step 2: Build Docker Image

Command:

```bash id="u7k3rm"
docker build -t mydocker .
```

### Explanation

This command:

* reads Dockerfile
* downloads Ubuntu image if not present
* creates a new Docker image
* names it `mydocker`

### Meaning of Parts

| Part           | Meaning                              |
| -------------- | ------------------------------------ |
| `docker build` | Create image                         |
| `-t`           | Tag/name the image                   |
| `mydocker`     | Image name                           |
| `.`            | Current folder containing Dockerfile |

### Output Meaning

```text id="x1n5qp"
Successfully built
Successfully tagged mydocker:latest
```

means image creation successful.

---

# Step 3: Display Docker Images

Command:

```bash id="z9m2vl"
docker images
```

### Explanation

Shows all Docker images stored in system.

### Output

```text id="v4q8wr"
mydocker
ubuntu
```

means:

* Ubuntu base image exists
* Custom image `mydocker` created successfully

---

# Step 4: Run Docker Container

Command:

```bash id="k6x1pn"
docker run mydocker
```

### Explanation

This command:

* starts a container from `mydocker` image
* runs Ubuntu environment
* executes CMD instruction

### Output

```text id="r7w3qy"
Docker Container Environment Created
```

This output is printed from inside the container.

After printing, container stops automatically.

---

# Step 5: View Containers

Command:

```bash id="m3v9lk"
docker ps -a
```

### Explanation

Displays all containers:

* running
* stopped

Docker automatically gives random names to containers.

---

# Step 6: View Logs

Command:

```bash id="t8q2wr"
docker logs CONTAINER_ID
```

### Explanation

Shows output generated inside container.

Example:

```text id="p5x7mn"
Docker Container Environment Created
```

# Important Viva Questions

## What is Docker?

Docker is a platform used to create and run containers.

---

## What is a Container?

A container is a lightweight isolated environment used to run applications.

---

## What is Dockerfile?

Dockerfile is a text file containing instructions to build Docker images.

---

## What is Docker Image?

Docker image is a template used to create containers.

---

## Difference Between Image and Container?

| Image    | Container        |
| -------- | ---------------- |
| Template | Running instance |
| Static   | Active/running   |

---

## Why Ubuntu Used?

Ubuntu provides a Linux base environment for the container.

---

## What Does `docker build` Do?

Creates Docker image using instructions from Dockerfile.

---

## What Does `docker run` Do?

Starts a container from an image.

---

Dockerfile → Build Image → Run Container → Output Generated
